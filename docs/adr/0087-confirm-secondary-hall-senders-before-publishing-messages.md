# 二级大厅按命令语义读取发送者

二级大厅视觉观察先用轻量像素结构检测深色气泡，再以圆形头像边界和局部纹理确认发送者起点。每个头像生成固定的发送者 OCR 区域：`x=410`、头像顶部上移 `10px`、高度 `36px`，最大文字宽度按 `18` 个全角字符计算并加左右各 `8px` 留白，总宽 `394px`。从该区域底部到下一个头像区域顶部之间的气泡归属于当前头像；连续气泡共享同一区域，新头像终止上一发送者。视觉观察不再依赖昵称或备注颜色像素分割。

气泡像素和固定发送者区域指纹共同构成低成本视觉身份，并按 `stability.secondary_hall_count` 局部覆盖或全局默认次数连续确认后才 OCR 正文。正文先使用固定的“二级大厅”身份做一次无副作用命令路由，再根据路由结果决定是否 OCR 发送者。点歌、播放器控制、帮助、大厅检测与时间，以及各玩法的纯查询或全局重试不读取昵称；报名、开局、接龙提交、出牌、投票、描述、退出等会改变玩家状态、权限或投递目标的输入才读取固定发送者区域。海龟汤问题同样依赖提问者，因此继续读取并稳定确认昵称。自定义流程把触发用户暴露给模板和默认好友目标，按身份相关命令处理。

无法路由的普通聊天只用于推进可见气泡序列基线，不发布为二级业务消息。身份无关命令以固定的“二级大厅”作为命令元数据；身份相关命令仍优先使用最右侧完整半角或全角括号对中的非空昵称。需要发送者而 OCR 为空时，本轮保持待确认并保留旧基线，不使用固定身份代替真实玩家执行。

生产路径不引入 OpenCV。根目录实验项目中的 OpenCV 实现只用于离线对照；生产采用其经过回归语料验证的轻量纯 Rust 检测边界。这样把误归属风险封闭在观察模块内，同时保留低成本常驻监控和可配置稳定次数。

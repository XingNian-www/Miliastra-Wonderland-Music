# 独占决策读者与共享命令流并行

候选确认、管理投票等独占决策会话拥有自己的消息基线和结果，读者在自己的轮询中只接受符合当前决策条件的新消息。`begin_exclusive` 只登记一个活动决策读者，不把观察值改道到私有流；会话期间的聊天观察仍然发布到共享命令流，普通命令继续由一级/二级监听按观察顺序解析并进入正式任务队列；一级使用命令屏幕锁，二级使用气泡序列/消息基线，二者都由正式任务去重键继续做语义去重。

独占读者不会把自己的决策结果再次提交为普通命令，也不会在会话结束时制造 `ExclusiveSession` 缺口。这样确认窗口中的普通命令不会因为独占等待而丢失，确认前已经可见的决策消息仍由 `DecisionScreenLock` 排除。监控可以展示脱敏摘要，但不能使用独占会话结果触发其他业务。

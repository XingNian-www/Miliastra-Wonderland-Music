# 好友投递只重试确认未发送的剩余项

共享好友投递契约使用一个非负整数统一配置每个确认未发送项的最大自动重试次数，`0` 表示禁用，纵向业务模块不能分别覆盖。活动娱乐玩法中的人工 `#重试` / `＃重试` 由当前玩法接收，并形成关联原批次标识和业务会话代数的新业务操作；非娱乐投递不提供全局重试命令，只返回失败原因。两种重试都只能继续确认未发送和尚未尝试的剩余项，已完成项永久跳过，结果未知项不得自动或人工重放。恢复驻留界面后，系统只对具备安全剩余项的娱乐失败提示 `#重试`，结果未知则返回明确原因并等待取消或其他人工处理。接受部分失败无法通过重发自动修复的成本，是为了避免重复发送手牌、卧底词或其他私密内容，同时避免 UI 自动化内核拥有业务命令语义。

# 旧版单体启动千星流程

## 状态

已由 ADR 0068 取代。本文件仅保留旧方案的背景与迁移原因；当前实现不再提供“启动千星流程”这一单体任务。

## 背景

BetterGI 的相关实现包含两个方向：

- 自动进入游戏，也就是开门
- 进出千星奇域循环，用于进入大厅后再返回提瓦特

本项目需要的是无人值守启动到千星大厅，用于后续音乐机器人运行，不需要把千星奇域当作刷新工具反复进出。

## 决策

历史方案曾新增“启动千星流程”，作为主业务队列中的高层任务执行：

1. 查找官服/国际服游戏窗口
2. 找不到窗口时启动游戏
3. 聚焦游戏窗口一次
4. 用 OCR 点击“进入游戏”和必要确认按钮
5. 周期性按 `F6`，等待千星奇域界面出现
6. 点击配置的奇域卡片坐标
7. 用 OCR 点击“前往大厅/进入大厅”
8. 等待一级聊天界面的回车模板出现

流程到达千星大厅后结束，不自动返回提瓦特。

第一版只支持官服和国际服通用路径：

- `YuanShen.exe`
- `GenshinImpact.exe`

不处理 B 服 bilibili 协议窗口或登录窗口。

## 后果

- 启动千星流程和点歌、邀请、控制台发言共享同一个待执行任务队列，避免并发抢屏幕输入。
- HTTP 面板只提交远程任务，不直接操作窗口。
- 进入千星后的成功判断复用现有一级聊天界面回车模板，减少新增模板资产。
- 开门和千星按钮优先使用 OCR 文本框中心点击，因此不依赖 BetterGI 的 GPL 模板资产。

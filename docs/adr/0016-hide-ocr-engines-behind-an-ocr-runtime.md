# 通过 OCR 运行时隐藏识别引擎

OCR 推理由深层运行时模块独占。调用方提交已经裁剪并带稳定标识的图像和类型化识别请求，运行时返回结构化文本、置信度和局部区域结果。截图、游戏坐标、区域选择和裁剪留在 OCR 之外；批量图像拼接只是运行时内部优化。默认实现拥有一个生产识别引擎，依次按 UI 确认、聊天观察、后台观察和诊断的优先级调度；由画面变化触发但已经过期的观察请求可以在推理前合并。原始 OCR 引擎及其同步机制不能越过运行时接口；模型和后端配置属于 OCR，聊天切块、画面变化检测和稳定策略属于使用 OCR 的观察模块。未来可以在不改变接口的情况下把单引擎替换为引擎池。

OCR 运行时负责推理失败，观察模块负责画面不稳定和重复采样，UI 例程负责目标级确认。业务模块只接收已完成、超时、已取消或不可恢复的结果，不自行实现 OCR 轮询。

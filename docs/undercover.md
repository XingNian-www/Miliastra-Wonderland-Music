# 谁是卧底

谁是卧底默认关闭。启用后，它与成语接龙、斗地主、海龟汤互斥，同一时间只能进行一种娱乐玩法。

## 开局

- 单卧底使用公屏 `#卧底` 创建房间，4 到 11 人。
- 双卧底使用公屏 `#卧底双` 创建房间，6 到 11 人；两名卧底拿到同一个词，但不知道对方是谁。
- 其他玩家只能在好友私聊发送 `#加入` 报名。满 11 人自动开始；达到最低人数后，创建者可在公屏发送 `#开局`。
- 创建或加入时，好友列表昵称必须达到连续稳定次数才确认报名。`invite.friend_name_stable_count` 大于 1 时覆盖全局值，0 或 1 继承 `stability.default_count`，两者均无有效值时使用内置默认 2 次。
- 开局时每人随机得到固定的 A 到 K 位置。机器人只在好友私聊中发送“位置”和“词语”，不会告诉玩家身份。
- 发词失败会重试一次；仍失败则取消本局。已经选中的词对不会再次使用。

卧底控制命令忽略空格，例如 `#状 态` 与 `#状态` 等价。

## 描述和投票

1. 每位存活玩家在公屏发送 `#内容` 完成描述。描述最多 70 显示宽度，不能直接写出自己的完整词语，提交后不能修改；机器人只记录完成状态，不转发描述正文。
2. 全部描述收齐后立即进入投票。
3. 投票时请好友私聊 `#怀疑对象编号`，例如怀疑 A 号就发送 `#A`；也可以发送 `#弃权`。弃权计入已投票但不增加任何人的票数，提交后不能修改。
4. 最高票玩家出局并公布身份，但不公布词语，然后进入下一轮。

## 局外谜底查询

- 当前对局外的好友可以私聊 `#谜底`，机器人只向该好友发送一条紧凑结果：`平民词:XXX,卧底词:YYY,卧底:AB`。
- 当前对局玩家和已经淘汰的对局玩家都不能使用 `#谜底`；大厅发送同名文本不会触发谜底查询。
- 查询是只读操作，不提交描述或投票，不改变阶段、轮次、玩家状态或任何超时计时。

投票阶段每 20 秒公布一次尚未投票的位置。描述或投票阶段 180 秒无输入时按当前阶段结算：未描述者视为退出，未投票者视为弃权。整局连续 3 分钟没有有效命令会以无胜方结束。

若本轮收到的全部都是弃权票，则无人淘汰并进入下一轮；只要存在有效对象票，仍按对象票中的最高票结算。

平票时，并列玩家各补充一次描述，其他存活玩家再投一次。第二次仍平票则无人出局，直接进入下一轮。

## 胜负和退出

- 所有卧底出局：平民胜利。
- 存活平民人数不多于存活卧底人数：卧底胜利。
- 游戏中退出等同出局。报名阶段创建者退出会取消房间，其他人退出只离开报名。
- 创建者可用公屏 `#结束` 主动结束，结果为无胜方。
- 公屏 `#状态` 查看当前阶段。对局中的状态格式为 `单卧底,第1轮,发言,Abcd`：小写表示本轮未发言，大写表示已发言；投票阶段列出的玩家均已完成本轮发言，已淘汰或不参与当前加赛的玩家不列出。`#退出` 退出，报名玩家也可在好友私聊使用 `#退出`。

结算先公布胜方和两个词，再按每行 3 人公布位置、昵称和身份，最后公布总轮数。

## 词库和 Web

复制 `undercover.example.yaml` 为 `undercover.yaml` 后填写词对，再把 `config.yaml` 中的 `undercover.enabled` 改为 `true`。`A:B` 和 `B:A` 被视为同一词对；词对在第一次发词前永久标记为已用，即使之后取消也不会恢复。使用记录损坏时会拒绝开局并保留报名房间。

Web 面板可查看脱敏状态、开始已满足人数的报名房间、结束房间或对局。页面不会显示词语、身份、描述或投票，也不能代替玩家报名、描述或投票。

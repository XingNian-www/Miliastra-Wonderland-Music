# AI 调用统一使用 OpenAI 标准协议与 SDK

Rust 主程序的点歌和海龟汤调用使用标准 Chat Completions，候选歌曲审核使用标准 Responses、`web_search` 与严格 JSON Schema；云崽插件使用官方 Node `openai` 包调用 Responses。Rust 没有 OpenAI 官方 SDK，因此采用基于官方 OpenAPI 规格的 `async-openai` 社区包。共享 transport 只处理完整 endpoint、Bearer 或 `api-key` 鉴权、连接复用、超时和标准错误，不拥有业务提示词、解析或重试。`OpenAiRuntime` 由应用组合入口显式拥有，业务模块只保存可克隆的 handle；提交请求返回 operation，命令执行线程、HTTP 的 `spawn_blocking` 处理器和海龟汤专用 worker 才等待结果，不阻塞业务 runtime 线程。应用停止时先停止这些消费者，再显式关闭并等待 AI runtime。SDK 默认重试会改变费用、时序和各业务已有重试次数，所以 Rust 显式使用无重试的 `ReqwestService`，Node 显式设置 `maxRetries: 0`。供应商私有请求字段默认不进入生产请求；确有兼容需要时只能由对应 Provider 的 `extra_body` 显式补充，且 OpenAI 官方字段在同名冲突时优先。Provider 不支持对应标准 API 时应明确失败，由所属业务沿用自己的失败策略。

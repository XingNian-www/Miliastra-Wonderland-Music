# 米游社大世界音乐自动化

这个上下文描述本项目如何把游戏界面的机械操作组合成更高层的自动化流程。

## 语言

**原子动作**：
不可再拆的机械游戏界面操作，例如等待、按键、点击坐标、等待模板、点击模板、等待 OCR 文本、点击 OCR 文本、粘贴文本。
_避免使用_：工作流步骤、业务流程

**UI 自动化内核**：
集中承载游戏界面机械交互规则的深模块集合，为业务流程提供少量稳定的 interface，并统一处理视觉定位、输入安全、等待和失败语义。
_避免使用_：工具函数集合、业务流程层、文件拆分层

**UI 运行时**：
独占游戏窗口截图、输入和 UI 例程执行顺序的运行时所有者。所有游戏窗口交互都经它串行处理；OCR 推理和播放器操作由各自的运行时所有者负责。
_避免使用_：命令执行线程、屏幕锁、OCR 运行时

**业务等待**：
业务状态等待玩家输入、计时期限或外部结果的阶段；它不占用 UI 运行时，后续事件到达时再产生新的短 UI 工作。
_避免使用_：界面等待、线程睡眠、长 UI 例程

**挂起业务操作**：
已经提交异步 UI、AI、播放器或计时请求，并保存等待结果状态的业务操作。它可以继续占据正式任务顺位，但不能阻塞业务运行时处理新事件。
_避免使用_：阻塞等待、业务等待线程、已完成任务

**界面等待**：
UI 例程在一次界面输入后等待动画、模板、OCR 目标或像素稳定的确认阶段。它继续保持当前 UI 所有权，避免其他输入改变正在确认的界面。
_避免使用_：业务等待、娱乐计时器

**UI 占用策略**：
一次 UI 例程对 UI 运行时的占用约定。默认独占至完成；只有经过明确批准的例程才能在声明的安全点返回进度并让位，固定秒数不能作为占用边界。
_避免使用_：业务等待时间、任务优先级、任意中断

**可让位白名单**：
经过明确确认、允许在指定安全点释放 UI 运行时的 UI 例程集合。白名单随源码和行为测试维护，不能通过运行时配置扩大。
_避免使用_：可配置中断、默认可让位、按时间抢占

**UI 例程**：
由多个原子动作组成、可在 UI 自动化内核中复用的可靠界面交互，例如打开唯一好友聊天或恢复当前大厅。它隐藏交互顺序、稳定确认和重试，但不一定直接成为业务模块可调用的契约。
_避免使用_：原子动作、业务流程、工作流步骤

**UI 例程契约**：
业务模块可调用的一次完整机械事务所拥有的类型化请求与结果约定。它从声明的界面前置条件开始，独占完成目标输入并确认稳定结束状态后才返回；只留下中间界面的内核子例程不能单独暴露，不同契约也不被压缩成统一的万能结果。
_避免使用_：通用 UI 请求枚举、原子动作计划、业务结果、中间界面句柄

**UI 驻留目标**：
业务调度方为一次 UI 例程契约显式选择的稳定结束界面，例如一级界面或二级当前大厅。目标来自该契约允许的封闭集合；UI 自动化内核不读取聊天监听模式，也不恢复任意调用前画面。
_避免使用_：监听模式、调用前画面、任意界面快照、隐式恢复目标

**建立 UI 驻留**：
以确认到达指定 UI 驻留目标为唯一目标的完整 UI 例程契约，只供聊天监听模式切换和明确的独立恢复任务使用。其他业务不能把它作为目标动作前的准备步骤；完整例程只在自身事务内复用驻留导航子例程。
_避免使用_：通用界面准备、任务结束统一恢复、链式前置例程、`ensure_ui_residency`

**UI 起点归一化**：
公开 UI 例程在同一次独占事务内观察当前界面，并从自身支持的稳定界面转换到目标动作所需起点。未知或过渡画面只允许有限复检，仍无法判断时以输入前失败结束；业务模块不能先调用独立的界面准备例程。
_避免使用_：命令前统一准备、盲按 Esc、链式确保界面、业务层界面导航

**UI 例程执行上下文**：
UI 自动化内核在一次独占例程内提供的窄机械能力集合，包含窗口观察与输入、视觉定位、类型化 OCR 确认和时间控制。OCR 推理由独立运行时拥有，但例程等待识别结果时继续保持 UI 所有权；上下文不包含业务状态、监听模式或完整应用对象。
_避免使用_：全局应用上下文、OCR 引擎、业务能力容器、跨例程界面句柄

**UI 例程完成结果**：
UI 例程契约分别报告目标动作效果和结束驻留结果；两者都确认后才是完整成功。未完整成功时仍保留该契约特有的效果确定性与进度，例如已发送条数，不能压缩成通用错误或万能效果类型。
_避免使用_：`Result<()>`、恢复失败等于目标未执行、通用 Effect 枚举、错误文本推断进度

**UI 收尾恢复**：
目标动作进入成功、失败、部分完成或结果未知等终止状态后，UI 例程重新观察当前界面，并只从可确认状态有限收敛到请求中的驻留目标。收尾不能重复目标动作；窗口丢失或界面未知时停止输入，并把恢复结果与目标动作结果分别报告。
_避免使用_：任务外统一按 Esc、失败后重放、忽略目标动作进度、盲目恢复

**UI 例程进度**：
长 UI 例程按操作标识发布的脱敏只读阶段信息，例如当前收件人序号、消息序号或收尾阶段。进度不包含昵称和消息正文，不是业务结果、让位点或取消能力；最终结构化结果仍是唯一业务事实。
_避免使用_：敏感日志、业务状态、可中断检查点、结果回调

**UI 失败确定性**：
UI 例程失败时对游戏输入影响的结构化说明，区分输入前失败、输入后结果未知、已确认失败和已取消，并携带失败阶段。它决定是否存在安全重试的可能，不能由错误文本推测。
_避免使用_：通用错误字符串、默认可重试、HTTP 状态

**UI 例程重试**：
具体 UI 例程在确认尚未产生输入或拥有幂等恢复方式时执行的有限机械重试。业务模块只能接收例程最终结果，不能把整次例程失败隐式转化为重新执行。
_避免使用_：业务重试、结果未知后重试、用户新请求

**工作流步骤**：
自定义工作流配置中的一个顺序节点。它要么属于可组合原子动作的机械步骤，要么是交给拥有模块执行并允许工作流挂起的类型化能力步骤；两者不能在同一个 UI 动作计划中嵌套执行。
_避免使用_：原子动作、通用 Effect、内置业务状态机

**自定义动作计划**：
自定义工作流解释器把一段连续机械步骤解析并校验后形成的 UI 执行计划。它是唯一允许直接组合原子动作的入口，并作为不可插入其他输入的独占 UI 例程执行；遇到类型化能力步骤时必须结束当前计划。
_避免使用_：内置业务流程、通用 UI 例程、逐原子动作入队、完整业务工作流

**工作流机械段**：
自定义工作流中一组连续机械步骤形成的独占自定义动作计划。机械段完成后才能调用类型化能力或开始下一段，段内等待不会释放 UI 所有权。
_避免使用_：业务等待、单个原子任务、可插入动作序列

**工作流能力步骤**：
自定义工作流交给对应纵向业务模块或类型化 UI 契约执行的高层节点，例如好友投递或邀请。工作流保存进度并在 UI 运行时之外等待关联结果，完成后再恢复后续步骤；能力步骤不能在机械段内部嵌套提交。
_避免使用_：原子动作、嵌套 UI 例程、通用业务效果、内置工作流引擎

**业务流程**：
更高层的游戏自动化行为，例如邀请用户、管理用户、发送好友消息、返回主界面。
_避免使用_：原子动作、工作流步骤

**像素稳定**：
对同一个游戏区域连续截图时，视觉变化足够低，可以认为界面过渡已经结束的状态。
_避免使用_：模板消失

**文本剪贴板临时占用**：
粘贴文本时临时写入系统文本剪贴板，完成粘贴后尽量恢复原文本内容。
_避免使用_：无剪贴板粘贴、完全恢复剪贴板

**游戏聚焦前置条件**：
原子动作执行前游戏窗口已经处于可接收输入的状态；聚焦本身由业务流程或显式前置动作负责。
_避免使用_：每个原子动作自动聚焦

**安全聚焦点**：
用于让游戏窗口接收键盘输入的通用点击点，不承载聊天输出或返回主界面的业务含义。
_避免使用_：输出聚焦点、聊天聚焦点

**点击归属校验**：
点击类原子动作在发送鼠标输入前确认目标屏幕点属于游戏窗口，避免被遮挡窗口或错误坐标截获。
_避免使用_：自动聚焦校验

**前台输入校验**：
非点击输入类原子动作在发送按键或粘贴前确认当前前台窗口属于游戏进程。
_避免使用_：点击归属校验

**控制台发言**：
远程面板向游戏聊天频道排队发送的一条可见文本，文本会带有固定的控制台来源标识。它不是玩家命令，不参与游戏内命令解析。
_避免使用_：模拟命令、远程伪造聊天命令

**远程指挥台**：
远程查看状态并提交高层意图的 Web 面板。它不直接暴露点击、按键、粘贴等底层游戏界面操作。
_避免使用_：远程鼠标键盘、远程原子动作面板

**AI 自动化基础客户端**：
供 AI 单独调用现有 Web 工具能力的外部客户端，只执行截图、观察或单个输入操作，不负责编排完整自动化流程。
_避免使用_：自动化工作流引擎、远程指挥台、无人值守测试系统

**远程点歌**：
从远程指挥台提交的点歌意图，语义上等同于游戏内点歌请求。它必须与游戏内点歌共享同一套排队、保护和反馈规则。
_避免使用_：HTTP直连播放、后台强制切歌

**控制台反馈**：
由远程指挥台触发的业务流程在游戏聊天中留下的可见反馈。反馈必须带有控制台来源标识，避免被误认为玩家发言。
_避免使用_：静默远程操作、无来源反馈

**远程任务**：
远程指挥台提交到主业务队列中的高层意图。HTTP 请求只确认远程任务已入队，执行结果通过监控、日志和控制台反馈观察。
_避免使用_：同步远程调用、HTTP直接执行

**HTTP 业务适配器**：
负责鉴权、协议字段校验以及 HTTP 请求与类型化业务意图或查询之间转换的边界。它不解析聊天命令，也不直接修改业务状态、播放器或游戏窗口。
_避免使用_：HTTP 业务服务、伪造聊天命令、直接状态访问

**远程播放控制**：
从远程指挥台提交的暂停、继续、上一首、下一首和音量意图。它必须映射成游戏内同名业务命令进入主业务队列，不能直接绕过队列调用播放器。
_避免使用_：HTTP直接切歌、远程播放器快捷键

**点歌互斥**：
所有来源的点歌请求共享同一个执行保护边界，远程点歌不能绕过游戏内点歌，游戏内点歌也不能绕过远程点歌。来源只影响反馈文案，不影响互斥语义。
_避免使用_：远程独立点歌锁、按入口分开的点歌保护

**命令屏幕锁**：
对当前仍可见在游戏聊天屏幕内的同语义玩家命令做一次性执行保护。它防止旧 OCR 文本重复入队，不代表业务互斥或音乐播放保护。
_避免使用_：点歌锁、当前歌曲保护、待执行任务队列

**同语义命令**：
在业务含义上应视为同一次请求的命令，即使 OCR 原文、大小写、标点或前缀略有差异。它用于去重，不代表两个命令的文本完全相同。
_避免使用_：相同文本、重复OCR

**命令信封**：
聊天入口交给业务路由的结构化上下文，包含观察消息标识、昵称、来源、原文和权限信息，但不包含任何具体业务命令类型。
_避免使用_：统一业务命令、OCR 消息、HTTP 请求

**命令路由器**：
根据命令信封和当前模块占用选择唯一纵向业务模块的静态应用边界。确认、跳过、换源、邀请和投票回复等决策语法由它保留给决策读者，不会被自定义工作流抢占；除此之外它不解析业务参数，也不执行业务规则。
_避免使用_：中央命令解析器、动态插件系统、业务调度器

**启动屏幕锁**：
程序刚启动或进入新视觉会话后，先记录当前可见命令但不执行的保护机制。它避免启动前残留消息被当成新命令。
_避免使用_：命令屏幕锁、启动互斥

**确认屏幕锁**：
在等待 `@确认`、`@跳过`、`@换源` 等决策消息时，忽略等待开始前已经可见的确认消息。它只服务当前确认窗口，不影响普通命令入队。
_避免使用_：命令屏幕锁、全局命令锁

**决策命令**：
在候选确认、换源、AI 自动匹配等等待窗口里回复的短命令，例如 `@确认`、`@跳过`、`@换源`、`@AI`。这些语法只交给当前决策读者；独占等待期间出现的其他普通命令仍进入共享命令流，并按命令锁和待执行队列去重。
_避免使用_：普通命令、管理命令

**聊天区变化指纹**：
用于判断聊天区域是否发生视觉变化的低分辨率灰度摘要。它只决定是否触发聊天扫描，不代表已经识别出新消息。
_避免使用_：OCR结果、聊天消息

**聊天监听模式**：
运行期决定系统从哪一种聊天界面读取玩家消息的状态。它可由远程指挥台或好友私聊命令切换，不属于启动配置。
_避免使用_：聊天扫描配置、聊天界面配置

**聊天监听命令**：
用于切换或查询聊天监听模式的高级命令：`@监听模式 一级`、`@监听模式 二级`、`@监听模式 状态`。仅好友私聊和控制面板/API 可以发起；当前大厅消息不能切换监听模式。切换命令进入待执行任务队列，避免与其他界面动作抢占屏幕；状态命令只读取当前模式和等待状态，不操作游戏界面。远程指挥台展示当前模式和切换等待状态，并提供一级、二级切换按钮；TUI 只展示状态。
_避免使用_：大厅管理命令、启动配置项

**一级监听**：
从游戏一级聊天区读取消息的聊天监听模式。它使用聊天标记区分大厅消息和好友私聊消息。
_避免使用_：默认扫描、主界面扫描

**二级监听**：
从游戏二级聊天界面读取消息的聊天监听模式。切换进入该模式时，程序复用发送消息的前半段：聚焦游戏后按 Enter 打开二级聊天界面，点击左侧“当前大厅”模板，并确认顶部身份已经是“当前大厅”，再建立消息基线。整套进入流程最多尝试两次；第二次开始前先按 Esc 回到一级界面。两次均失败时恢复一级监听。它根据当前聊天身份把消息映射为大厅消息、好友私聊消息或忽略。
_避免使用_：好友面板扫描、二级配置

**二级好友会话**：
二级监听中通过好友未读红点进入、且当前顶部标题不属于已知非好友关键词的会话。它等价于好友私聊来源，允许全部现有好友私聊命令；不要求 OCR 准确识别该好友昵称。
_避免使用_：受限好友模式、二级私聊特例

**当前聊天身份**：
二级聊天界面顶部标题提供的会话分类线索。识别到“当前大厅”映射为大厅消息；识别到“公开频道”或“陌生人消息”视为非好友会话；其他标题内容一律映射为好友私聊。它不用于验证具体好友昵称。
_避免使用_：当前频道、聊天标题

**二级监听回退**：
二级监听遇到不可执行的当前聊天身份时，先尝试切回当前大厅；如果无法切回，再恢复到一级监听。
_避免使用_：忽略频道、自动退出聊天

**二级大厅恢复**：
二级监听来源的普通命令执行后的目标状态。它先检测是否仍处于二级聊天界面：若仍在二级界面，只点击“当前大厅”并确认顶部标题；若命令已离开二级界面，才通过一级界面重新打开二级聊天并切回当前大厅。它不能无条件先回一级界面。
_避免使用_：固定回一级、重复打开聊天

**邀请会话复用**：
邀请已被同意、默认同意或确认当前为公共大厅后，在同一个独占 UI 机械事务内确认目标好友、发送通知并继续邀请步骤。好友会话只能在该事务内部复用，不能通过布尔值或句柄跨 UI 例程保留；大厅邀请决策等待仍位于事务之外。
_避免使用_：`keep_chat_open`、跨例程好友会话、邀请等待占用 UI、无条件回一级

**邀请通知**：
邀请 UI 事务在确认目标好友后发送的尽力私聊提示。通知失败会保留原因但不阻止后续邀请；邀请完成后只在驻留大厅报告警告，不重新打开旧好友会话重试。
_避免使用_：邀请前置条件、可人工重试投递、邀请结果、静默通知失败

**稳定好友候选**：
出站好友操作在好友列表中连续确认的唯一目标，由完整规范化昵称和同一列表行位置共同标识。列表滚动、行位置超出容差、多个命中或命中消失都会重置稳定进度；不能只累计任意单一昵称命中的次数。
_避免使用_：昵称命中次数、最后一次 OCR 框、当前聊天身份、模糊好友行

**好友列表遍历**：
出站好友操作只在严格好友列表区域内执行的受限翻页查找。每次滚动后必须等待列表稳定并建立页面指纹，页面重复或达到上限时停止；滚动期间不能点击，也不能沿用上一页的候选坐标。
_避免使用_：全屏昵称 OCR、无限滚动、旧坐标点击、滚动中匹配

**好友投递批次**：
一次不可插入的 UI 机械事务中按顺序处理的一个或多个好友及其消息列表。每位收件人到达时才查找并确认，不在发送前预检全部好友；批次只在全部投递结束后恢复一次驻留界面，并分别记录每位好友的投递进度。已完成或结果未知的目标不能由同一请求重发。
_避免使用_：逐好友 UI 任务、可插入私聊发送、无进度群发、失败后整批重发

**好友投递重试**：
对失败好友投递批次中确认未发送和尚未尝试部分的继续投递。共享好友投递契约使用非负整数统一配置每个确认未发送项的最大自动重试次数，`0` 表示禁用；活动娱乐玩法把人工 `#重试` / `＃重试` 作为关联原批次和会话代数的新业务操作，非娱乐投递只返回失败原因。已完成、部分完成中的已发送项以及结果未知项都不具备重试资格，纵向业务模块不能覆盖自动重试次数。
_避免使用_：整批重发、结果未知后重试、普通任务重新入队、丢失原批次身份

**等待投递重试**：
活动娱乐玩法保存唯一一个可安全继续的失败好友投递批次，并暂停依赖该投递的阶段。该状态只由 `#重试` / `＃重试` 恢复安全剩余项，或由 `#结束` 取消；解决前不开始新的好友投递批次。
_避免使用_：投递重试队列、多个待重试批次、失败后继续阶段、无批次身份重试

**好友未读探针**：
二级监听中仅扫描左侧好友列表头像右上角窄区域的红色未读标记，用来发现未打开的好友私聊。红色候选还必须通过尺寸、白色感叹号、相对 `48px` 圆形头像的位置和头像边界校验，避免把红色头像内容当成未读；点击后在原标记附近复核红色像素是否消失。每次只处理一位带标记好友：进入会话、确认该行红点已消失、等待短暂稳定、只 OCR 最下方最新的他人深色气泡，然后恢复当前大厅并至少监听一轮，再处理下一位。顶部标题只排除已知非好友关键词，不校验具体昵称。首次进入二级监听时，当前可见的红点只会被逐个打开并清除，不 OCR、不解析、不入队；屏幕外好友不自动滚动查找，之后自然进入可见区时才按正常流程处理。它不扫描全屏红色元素，也不处理陌生人消息。
_避免使用_：全屏红点匹配、好友列表轮询、二级消息锁

**二级消息锁**：
二级监听中按当前聊天身份维护的可见消息去重边界。当前大厅保存完整可见且已关联头像锚点的气泡序列，用旧序列尾部与新序列头部的最长重叠找出全部新增气泡；只有存在可靠重叠时才 OCR 新增气泡，找不到重叠只重建基线。未初始化与有效空基线是不同状态，因此空大厅的第一条新消息仍会被处理。好友未读会话只使用最下方最新气泡。它避免切换会话或保留历史消息时重复执行旧命令，也避免同一屏连续大厅消息漏识别。
_避免使用_：命令屏幕锁、启动屏幕锁

**二级气泡探针**：
二级监听中用于低成本定位他人深色气泡的图像检测。当前大厅在固定客户区内以深色气泡左边缘作为锚点分段，定位每条气泡本体；好友未读会话定位最下方气泡。生产实现使用轻量像素结构检测，OpenCV 只作为离线实验对照。它不判断命令是否新增，也不替代二级消息锁。
_避免使用_：气泡消息锁、气泡命令识别

**二级发送者归属**：
当前大厅把圆形头像的边界与局部纹理作为发送者起点证据，并为每个头像建立右上方固定 OCR 区域。区域从 `x=410` 开始，头像顶部上移 `10px`，高 `36px`，宽度按最多 `18` 个全角字符加左右各 `8px` 留白计算。区域底部到下一个头像区域顶部之间的气泡归属于当前头像；连续气泡共享该区域，新头像截断上一发送者。正文先以固定大厅身份做无副作用路由，只有业务结果、权限、回合归属或投递目标依赖玩家的命令以及海龟汤提问才 OCR 固定区域；纯查询和控制命令不识别昵称。结果优先使用最右侧完整括号对中的名称，身份相关输入识别为空时等待下一轮，不以固定大厅身份代替真实玩家。
_避免使用_：按命令前缀一律 OCR 昵称、昵称或备注颜色分割、仅亮度头像、身份相关输入匿名执行、通用轮廓真值

**OCR 切块**：
在真正调用 OCR 前，根据聊天标记把聊天区域切成单条消息文本块的过程。它不负责理解命令语义。
_避免使用_：命令解析、OCR识别

**OCR 推理所有权**：
OCR 运行时对生产识别引擎和请求顺序的独占控制。调用方只提交有优先级的 OCR 请求，不共享引擎或推理互斥。
_避免使用_：OCR 全局互斥、调用方持有引擎、并行抢占推理

**OCR 运行时**：
独占 OCR 引擎并通过类型化请求返回结构化识别结果的深模块。调用方不能取得原始引擎或推理互斥，也不负责引擎生命周期。
_避免使用_：OCR 工具函数、共享 OcrEngine、聊天观察模块

**OCR 请求**：
调用方提交给 OCR 运行时的一次识别意图，声明需要的结果形状而不暴露引擎操作。
_避免使用_：OCR 引擎锁、聊天观察配置、原始推理调用

**标识图块 OCR**：
批量识别调用方已分配稳定标识的多个图像块，并保证识别结果只返回给原图块。无法可靠归属的结果必须显式失败，不能猜测关联。
_避免使用_：OCR 识别标识、按返回顺序关联、歧义强制归属

**视觉定位模块**：
组合图像区域、模板匹配和 OCR 结果以定位界面目标的深模块。它不执行文字推理，也不决定业务流程。
_避免使用_：OCR 运行时、模板匹配算法、业务流程

**共享观察流**：
由画面变化驱动、可供多个功能独立读取的非独占聊天观察序列。同一观察结果不会因某个读取者处理而从其他读取者视野中消失。
_避免使用_：共享消息队列、独占 OCR、命令屏幕锁

**观察消息标识**：
聊天观察模块为一条稳定可见消息分配的会话内身份，由视觉会话、聊天身份和气泡顺序确定。OCR 修订保持同一观察身份；它只消除视觉重复，不判断业务语义是否重复。
_避免使用_：OCR 文本哈希、命令锁键、业务请求编号

**观察缺口**：
订阅者落后超过共享观察流有界历史后收到的明确连续性丢失结果。订阅者必须按自身用途重建基线，不能把缺失范围静默当作已处理。
_避免使用_：空消息、自动补执行、全局阻塞

**观察完成水位**：
聊天观察模块已经完成处理的最新观察帧单调捕获时间。业务期限只有在水位越过截止点后才能确认截止前输入已经全部得到结果。
_避免使用_：当前时间、OCR 完成时间、固定宽限时间

**观察终止失败**：
观察帧经过规定 OCR 重试后仍无法形成消息的最终结果。它不产生玩家输入，但会记录原因并允许观察完成水位继续推进。
_避免使用_：空消息、无限重试、静默丢帧

**敏感文本**：
谁是卧底词语、秘密投票、海龟汤汤底等不能进入普通日志或监控的文本值。它默认只显示固定脱敏内容，只有拥有对应业务结果的模块才能显式读取原文。
_避免使用_：普通字符串、日志后处理、全局隐藏文本

**独占观察会话**：
业务流程为了特定界面和消息基线而临时登记的独占决策读取范围，适用于流程会主动切换或占用聊天界面的情况。独占读者在自己的轮询中只接受符合当前决策条件的新消息，不从共享流游标消费；会话期间的聊天观察仍发布到共享命令流，由普通监听继续解析并按现有锁和正式任务去重键入队。会话结束时不人为制造观察缺口。
_避免使用_：暂停共享命令流、OCR 全局互斥、永久会话

**观察配置**：
决定一次聊天观察所覆盖界面范围、稳定要求和触发方式的规则集合。只有观察配置相同的读取需求才能复用同一观察结果。
_避免使用_：业务命令、OCR 引擎配置、聊天监听模式

**聊天观察模块**：
与 UI 自动化内核并列的深模块，负责把游戏聊天界面的变化整理成共享命令流，并为需要独立消息基线的业务提供独占决策读者。它只观察和分发结果，不发送键盘或鼠标输入；独占读者不会把普通观察改道到私有流。
_避免使用_：UI 自动化内核、聊天命令解析器、消息队列

**观察帧**：
UI 运行时发布给观察模块复用的不可变游戏画面，携带来源 UI 会话和检测到的界面状态。只有与观察配置匹配的观察帧才能产生聊天消息或玩家命令。
_避免使用_：原始截图、无上下文缓存帧、OCR 结果

**画面采样需求**：
聊天观察模块向 UI 运行时声明的空闲截图节奏与观察上下文。UI 运行时只负责满足需求并发布观察帧，变化检测、OCR 时机和稳定策略仍由观察模块决定。
_避免使用_：UI 轮询规则、OCR 请求、常驻截图线程

**帧界面状态**：
视觉定位模块针对同一观察帧统一计算并共享的公共 UI 状态。具体 UI 例程可以追加目标检测，但不能各自重新定义或改写该帧的公共状态。
_避免使用_：业务目标状态、重复 UI 检测、监控文本

**界面状态证据**：
帧界面状态的结构化判定依据，包括参与判定的模板、区域、分数、阈值、命中框、稳定进度和最终采用的规则。业务可以只读取状态，诊断和失败结果保留完整证据。
_避免使用_：状态字符串、调试截图、单一置信度

**状态稳定跟踪器**：
UI 运行时中根据连续观察帧累计公共 UI 状态的会话组件。视觉定位只提供单帧判定，状态稳定跟踪器按具体状态标签统一产生稳定进度和最终确认状态；同一帧不能重复累计。`unknown` 只表示过渡中，会重置候选进度但不授权输入、不改变监听模式，也不能用上一稳定状态代替当前状态。
_避免使用_：模板匹配器状态、调用方独立计数、目标确认器

**稳定次数继承**：
所有连续确认默认使用同一个全局稳定次数，具体 UI 状态、目标、昵称或正文识别可以显式覆盖。局部值大于 1 时优先使用，否则继承大于 1 的全局值；两者都未提供有效自定义值时使用内置默认 2。它只统一默认次数，不合并各自的阈值、超时或判定规则。
_避免使用_：强制统一次数、统一识别参数、固定次数

**纵向业务模块**：
围绕一项完整业务能力组织命令语言、状态规则和结构化结果的模块。它不拥有聊天观察、界面机械操作或运行时调度。
_避免使用_：技术分层工具模块、UI 自动化内核、主执行器分支

**业务应用服务**：
纵向业务模块中协调命令解析、领域状态和共享能力的边界。它把领域结果转换为聊天发送、UI 例程、计时或持久化调用，中央组合入口不再按业务类型翻译结果。
_避免使用_：纯领域状态、中央业务分支、UI 运行时

**能力端口**：
业务应用服务为完成自身职责所依赖的窄接口，例如聊天发送、UI 例程、时钟或存储。模块只获得实际需要的能力端口，不能访问包含全部共享对象的应用上下文。
_避免使用_：全局服务容器、AutomationApp 引用、万能依赖包

**结构化业务结果**：
纵向业务模块处理输入后返回的领域结果，表达该业务发生了什么以及需要哪些已知共享能力继续完成。不同业务可以拥有不同结果类型，不要求统一成万能动作集合。
_避免使用_：通用 Effect 枚举、UI 原子动作、日志文本

**应用组合入口**：
负责装配运行时、共享模块和纵向业务模块的唯一入口。它只建立所有权与依赖关系，不承载具体业务规则。
_避免使用_：主业务模块、万能控制器、程序入口函数

**业务调度器**：
在正式任务、延迟回复和诊断请求之间按既定优先级选择下一项可执行工作的应用层边界。它不执行 UI 例程，也不把业务种类暴露给 UI 运行时。
_避免使用_：UI 运行时、待执行任务队列、业务流程

**业务运行时**：
在单一专用线程中拥有命令路由、业务调度器、娱乐互斥和各纵向业务模块状态的运行时。观察、计时、AI、播放器和 UI 通过类型化消息与它交互，不能直接修改业务状态。
_避免使用_：每玩法一个线程、AutomationApp、共享业务互斥锁

**业务操作标识**：
业务模块提交一次异步 UI、AI、播放器或计时请求时分配的唯一标识，用于把完成结果送回原始操作。它不表示业务会话仍然有效。
_避免使用_：任务标签、命令锁键、观察消息标识

**业务会话代数**：
业务会话创建、替换或结束时变化的版本标识。异步结果只有同时匹配业务操作标识和当前会话代数时才能应用，否则视为过期结果。
_避免使用_：时间戳、牌局编号文本、隐式当前状态

**计时运行时**：
统一管理业务截止时间并向业务运行时发送到期事件的运行时所有者。它只理解操作标识、会话代数和单调期限，不包含娱乐玩法的提醒、续期或结算规则。
_避免使用_：玩法 tick 线程、系统时间定时器、业务状态机

**运行时所有者**：
在专用线程中独占并修改一组运行状态的模块，其他模块只能通过类型化消息提交意图或接收结果。
_避免使用_：共享全局状态、任意线程加锁修改、异步任务容器

**音乐播放队列**：
已经确定候选歌曲、等待合适播放时机的歌曲列表。它只保存歌曲，不代表待执行的游戏操作。
_避免使用_：待执行任务队列、命令队列、PendingTask队列

**播放器运行时**：
独占 FeelUOwn 阻塞连接并执行状态读取、播放、暂停、切歌和音量等底层操作的运行时。观测、控制和候选搜索分别使用有界执行通道；它包含播放器观测运行时，但不拥有音乐播放队列、BOT 请求归属、当前歌保护或自动出队规则。
_避免使用_：播放器控制器、播放业务模块、自动出队线程

**播放器观测运行时**：
负责播放器连接、重连、重复采样、过期判断、本地进度估算和稳定确认的运行时组件。它输出带证据的稳定播放观测，但不判断歌曲是否属于 BOT 请求或是否应该保护。
_避免使用_：播放器控制器、单次 status、播放业务状态

**快速播放观察需求**：
播放业务在命令下发后提交给播放器观测运行时的临时高频采样请求，携带业务操作标识和截止时间。多个需求合并采样，不各自启动轮询线程，也不包含播放成功条件。
_避免使用_：播放确认、独立监控线程、固定快速轮询

**稳定播放观测**：
播放器观测运行时根据连续原始样本确认的现实播放器状态，包含 URI 歌曲身份、播放状态、进度、时长、可靠性和采样证据。URI 为空时只能确认播放状态，不能形成歌曲身份；它不是 BOT 已确认播放状态。
_避免使用_：原始播放器响应、活动播放请求、当前歌曲保护

**播放器命令回执**：
播放器运行时对一次播放、暂停、继续或切歌 RPC 是否完成下发的结构化结果。它不表示现实播放器已经达到目标状态；发送后断连时结果未知且不能自动重发。
_避免使用_：播放成功、稳定播放观测、聊天反馈

**播放器歌曲身份**：
稳定播放观测中用于判断同一首歌曲的非空 URI。歌名、歌手、进度和时长只能作为展示或状态证据，不能作为歌曲身份或 URI 缺失时的兜底。
_避免使用_：歌名歌手匹配、播放器元数据、点歌关键词

**播放实例代数**：
同一播放器歌曲身份每次重新开始播放时变化的版本标识。URI 变化、停止后重新播放或同一 URI 的进度明显回到开头都会产生新实例；普通进度增长和短暂抖动不会。
_避免使用_：歌曲身份、播放次数统计、业务会话代数

**过期播放观测**：
播放器出现空 URI、请求失败或不完整样本后暂时保留的上一条稳定播放观测。它携带过期时长，默认最多保留 5 秒且可单独配置；过期期间不能确认新播放或触发自动出队，超过上限后歌曲身份变为未知。
_避免使用_：新鲜观测、原始失败样本、永久缓存状态

**当前歌曲保护**：
新点歌到来时保留当前应该继续播放的歌曲，把新请求放入音乐播放队列等待后续播放。它不是命令屏幕锁，也不是 OCR 去重机制。
_避免使用_：歌曲锁、点歌锁、屏幕锁

**音乐播放队列去重**：
把最终候选歌曲加入音乐播放队列前，检查队列里是否已经存在同一候选。它只保护队列内容，不读取实际播放历史，也不代表歌曲已经被播放过。
_避免使用_：长时间同歌去重、播放历史去重、点歌互斥

**长时间同歌去重**：
在直接播放前、加入音乐播放队列前或音乐播放队列出队前，根据最近一段时间内确认播放成功的历史限制同一首歌再次播放。它只在确认播放成功后写入历史，入队前检查只拒绝确定的近期重复候选。
_避免使用_：音乐播放队列去重、屏幕锁、当前歌曲保护

**同歌播放历史**：
长时间同歌去重使用的持久化记录，只在确认播放成功后写入。只有双方都存在且逐字相等的非空 URI 才视为同歌；URI 缺失或不一致时不按歌名、歌手或跨平台相似度兜底。
_避免使用_：运行状态、点歌播放标记、队列文件

**播放器后端**：
具体对接音乐播放器的适配层。它提供播放、暂停、恢复、查询状态、搜索候选等底层能力，但返回的播放状态只能视为不稳定观测。
_避免使用_：播放器控制器、播放状态机

**播放器控制器**：
把播放意图、播放器观测、暂停原因和队列推进规则收敛到一起的业务模块。它不负责聊天确认和 AI 判断，但负责维护确认后的播放状态。
_避免使用_：FeelUOwn客户端、播放工具函数

**播放观测**：
一次从播放器后端读取到的状态信号，可能延迟、缺字段或短暂不一致。它不是业务事实，不能单独决定清空活动播放请求或触发自动出队。
_避免使用_：确认播放状态、运行状态

**确认播放状态**：
播放器控制器基于播放意图和多次播放观测维护的业务状态，例如启动中、正在播放点歌、用户暂停、等待队列暂停、外部播放或未知。
_避免使用_：播放器原始状态、单次status结果

**活动播放请求**：
播放器控制器确认当前播放器正在处理或播放的一次点歌请求。它记录目标 URI、歌名、歌手、来源和开始时间，用于当前歌曲保护、自动出队和播放历史写入。
_避免使用_：点歌播放标记、播放状态缓存、当前歌曲锁

**播放确认**：
播放器控制器在发出播放意图后，通过目标 URI、进度、时长和稳定等待判断播放是否真正开始的过程。普通点歌要求观测到的非空 URI 与请求 URI 逐字相等；URI 缺失或不一致时不使用歌名、歌手或 AI 诊断兜底。
_避免使用_：播放命令发送成功、FeelUOwn ACK

**自动出队**：
当播放器停止、暂停在结尾或即将结束时，把音乐播放队列的下一项交回主业务队列播放的机制。它不是 FeelUOwn 的下一首。
_避免使用_：播放器下一首、直接切歌

**临近结束暂停**：
系统为了等待队列或待执行点歌接管，在当前歌曲接近结束时主动暂停播放的状态。它不是用户主动暂停。
_避免使用_：用户暂停、播放失败

**候选歌曲审核**：
在点歌搜索和匹配产生最终候选歌曲后、播放或入队前，对候选歌曲及其 URI 做出的允许或拒绝判断。它审核的是即将执行的具体歌曲是否适合舒缓、轻松、不吵闹的房间氛围，不审核原始点歌意图。审核使用独立 Responses Provider、标准 `web_search` 和严格 JSON Schema，并可配置独立 HTTP(S) 代理。
_避免使用_：点歌意图审核、命令审核

**审核硬拒绝**：
候选歌曲审核拒绝后，本次点歌直接结束，不播放、不入队，也不进入确认流程。系统会向触发来源反馈拒绝原因。
_避免使用_：审核警告、降级确认

**审核失败策略**：
候选歌曲审核服务不可用、超时或返回无效结果时采用的兜底规则。默认拒绝本次点歌，但可以显式配置为放行。
_避免使用_：审核重试策略、AI错误处理

**追加审核规则**：
用户配置的候选歌曲审核提示词片段，会追加到主要审核条件之后。它用于临时补充本地审核标准，不替换插件协议。
_避免使用_：完整提示词替换、自由格式提示词

**主要审核条件**：
用户配置的候选歌曲审核主规则，用于定义房间氛围、通过标准、拒绝标准和保守策略。它不负责定义 JSON 输出协议或联网搜索开关。
_避免使用_：追加审核规则、审核 Provider 配置、JSON 输出协议

**审核强度**：
候选歌曲审核返回的 1 到 10 级打扰强度，用于和本地阈值比较后决定是否通过。分数表达歌曲破坏当前房间氛围的程度，而不是推荐程度。
_避免使用_：推荐分、匹配分

**审核阈值**：
本地配置允许播放的最高审核强度。候选歌曲的审核强度小于或等于阈值时通过，大于阈值时触发审核硬拒绝。
_避免使用_：最低通过分、推荐阈值

**控制台免审**：
由远程指挥台以控制台身份提交的点歌请求不经过候选歌曲审核。它只豁免审核，不豁免主业务队列、点歌互斥和播放保护。
_避免使用_：远程点歌免队列、管理员直连播放

**控制台去重豁免**：
由控制台来源提交的点歌或直接队列项默认不受长时间同歌去重限制。它不改变主业务队列、音乐播放队列和当前歌曲保护，只影响实际播放前的播放历史检查。
_避免使用_：控制台免审、远程直连播放、队列插队

**点歌 AI**：
用于理解点歌意图、选择候选歌曲或辅助同曲判断的 AI 能力。它不决定候选歌曲是否允许播放。
_避免使用_：审核 AI、候选歌曲审核

**AI 候选选择**：
点歌 AI 从 FeelUOwn 搜索候选中选择一个最终 URI 的判断。它只能选择已有候选，不能编造歌曲或 URI。
_避免使用_：候选歌曲审核、AI 搜索

**同曲判断**：
用于 Web 诊断的独立同曲判断结果。它可以展示本地或点歌 AI 的诊断，但不参与普通播放确认、不会写入同歌历史，也不能用歌名、歌手或跨平台相似度替代非空 URI 的逐字匹配。
_避免使用_：候选歌曲审核、播放状态检测

**审核 AI Provider**：
候选歌曲审核专用的 OpenAI Responses Provider 配置。它独立于 AI 点歌和海龟汤的 Provider，并且必须支持标准 `web_search` 工具和严格 JSON Schema；`http_proxy` 与 `extra_body` 只作用于该 Provider。
_避免使用_：点歌 AI、通用 AI 配置

**AI 能力网关**：
纵向业务模块拥有的类型化 AI 接口，负责该业务的提示词、请求协议、响应校验、重试和失败策略。不同业务网关只复用阻塞 HTTP 执行基础设施，不共享万能 AI 请求类型。
_避免使用_：通用 AI 模块、HTTP 客户端、统一提示词服务

**原子文件存储**：
为业务仓库提供临时文件写入、落盘和原子替换的共享基础能力。它不解释业务内容，也不决定读取或写入失败时应开放还是关闭业务。
_避免使用_：业务仓库、通用数据库、失败策略

**审核重试**：
候选歌曲审核请求失败、超时或返回无效审核强度时，在进入审核失败策略前进行的有限次数重试。
_避免使用_：播放器重试、搜索重试

**审核反馈摘要**：
审核硬拒绝时发送到游戏聊天的短原因。完整审核原因保留在日志中，聊天反馈会限制长度，避免刷屏或复述敏感内容。
_避免使用_：完整审核原因、AI 原文回复

**点歌执行**：
把一个点歌意图处理成最终播放结果的完整决策链，结果只能是播放、进入音乐播放队列、拒绝、取消或重复。它包含候选搜索、确认、AI 候选选择、候选歌曲审核、重复判断、当前歌曲保护和反馈。
_避免使用_：歌曲搜索、播放器调用、审核流程

**原始远程意图**：
远程指挥台提交的未经 AI、搜索或播放决策处理的用户意图。HTTP 层只传递原始远程意图，业务线程负责后续解析和执行。
_避免使用_：预解析远程请求、HTTP侧搜索结果

**娱乐互斥**：
成语接龙、斗地主、跑得快、海龟汤和谁是卧底共享的全局占用。任一玩法从开局到完整结算期间都会阻止其他玩法开局；游戏窗口关闭或进入新大厅时会释放旧大厅的娱乐状态。
_避免使用_：命令锁、待执行任务锁、点歌互斥

**娱乐命令路由**：
根据娱乐互斥的当前占用，把普通 `#` 输入交给唯一活动玩法的规则。无活动玩法时只路由明确开局与帮助命令，不能让多个玩法依次试解析同一输入。
_避免使用_：首个解析成功、广播命令、通用娱乐解析器

**湖南跑得快**：
三人使用 48 张牌、每人 16 张的牌类玩法。移除大小王、三张 2 和一张 A；持黑桃 3 的玩家先出且首手必须包含黑桃 3；有牌能压时不能过牌；下家只剩一张牌时，出单张必须使用手中最大单张。连对至少两对，允许裸三张、三带一、三带一对、四带二和四带二对，不允许四带三；三张 A 是普通三张，只有四张同点数牌是炸弹。轮到玩家时，零种合法压牌立即自动过牌，一种合法出牌立即自动打出，两种及以上才等待玩家选择；这种强制行动不增加超时或托管次数。连续强制过牌时，如果下一家仍会自动行动，第一条过牌消息静默；两家均无牌可压时只发送一条合并领出消息。出牌、过牌和胜负公告使用紧凑格式，并按 40 个全角字符上限设计，昵称按最多 8 个全角字符计算。最先出完手牌者获胜且不计分。除开局命令外，它复用斗地主的报名、出牌、过牌、手牌查询、超时和托管业务流程。
_避免使用_：抢地主、底牌、地主、农民

**双卧底模式**：
至少六人参与的谁是卧底身份配置，两名卧底持有相同词语但互相不知道身份。两名卧底全部淘汰时平民获胜；场上平民数与卧底数相等时卧底获胜。
_避免使用_：卧底队、卧底同伴可见模式

**公开描述**：
谁是卧底描述阶段中，存活玩家直接在大厅发送 `#内容` 完成本轮发言。`#` 必须位于消息正文开头；机器人只记录对应位置已经描述，不复述正文。所有应描述玩家完成后立即进入秘密投票。
_避免使用_：私聊描述、批量转述描述

**玩家位置**：
谁是卧底开局时从 A–K 随机分配并公开绑定昵称的字母标识，整局保持不变且淘汰后不重排。描述进度、公开描述和投票均以玩家位置作为稳定引用。
_避免使用_：回合序号、动态座位

**卧底词对**：
谁是卧底本地词库中经过人工审核的一组平民词与卧底词，可带分类并独立启停。词对身份不区分两个词的顺序，因此 `A:B` 与 `B:A` 是同一词对；开局只从启用词对中随机选择，不由 AI 临时生成。
_避免使用_：随机生成词、AI词语

**已用词对**：
谁是卧底抽中后、首次发词前已经写入永久排除记录的卧底词对。即使后续发词失败并取消整局，该词对也不会再次进入候选集合。
_避免使用_：近期词对、失败后返还词对

**使用记录保护**：
谁是卧底永久使用记录无法读取或校验时禁止开局、但保留当前报名房间的保护状态。它不会把损坏记录视为空记录，也不会放宽永久排除保证。
_避免使用_：记录失败后继续、自动清空黑名单

**卧底房间**：
等待谁是卧底玩家报名的临时会话。单卧底允许 4–11 人，双卧底允许 6–11 人；达到最低人数后创建者可开局，满 11 人时自动开局。
_避免使用_：卧底大厅、固定人数房间

**秘密投票批次**：
谁是卧底的一轮淘汰投票集合。存活玩家通过好友私聊各提交一张不可修改且不能投给自己的位置票；投票阶段每二十秒在大厅列出尚未投票的位置，全部收齐后统一公开得票汇总和淘汰结果。
_避免使用_：公开跟票、实时票数

**并列加赛**：
秘密投票最高票并列后仅进行一次的额外描述与限定重投，并列者不参与重投。加赛仍并列时本轮无人淘汰并进入下一轮公开描述。
_避免使用_：无限重投、随机淘汰

**淘汰公开**：
谁是卧底玩家被淘汰后立即公开其平民或卧底阵营，但保留具体词语。两组词与所有玩家身份只在整局结算时统一公开。
_避免使用_：淘汰后公开词语、全程隐藏阵营

**卧底胜利平衡点**：
场上存活平民数与存活卧底数相等的状态，单卧底和双卧底模式均在到达该状态时立即判定卧底获胜。
_避免使用_：剩余三人规则、卧底人数超过平民

**发词确认**：
谁是卧底开局前确认每位玩家已经私下收到固定位置和词语、但未被告知阵营的送达结果。单名玩家首次送达失败时重试一次，再次失败则取消整局。
_避免使用_：公开发词、私聊告知阵营

**诚实描述假设**：
谁是卧底参与者被视为不会刻意通过拆字、谐音或暗示绕过描述规则。系统只拒绝明显包含本人完整词语的描述，不承担语义作弊审查。
_避免使用_：AI描述审核、对抗式防作弊

**自愿淘汰**：
谁是卧底进行中玩家主动退出后形成的淘汰结果，公开阵营并继续检查胜负，不取消其他人的牌局。报名阶段普通玩家退出只移出房间，创建者退出才取消房间。
_避免使用_：单人退出整局取消、静默离场

**强制结束**：
谁是卧底创建者主动终止房间或进行中牌局的无胜方结算。进行中强制结束会公开两组词和全部身份，但不判定平民或卧底获胜。
_避免使用_：创建者退出、卧底胜利、平民胜利

**空闲结算**：
谁是卧底正式对局连续三分钟没有有效游戏命令后触发的无胜方结算。正式对局没有总时长上限；描述阶段部分玩家已提交时，未提交者仍按阶段截止规则自愿淘汰。
_避免使用_：单局总时限、零票进入下一轮

**卧底有效活动**：
当前存活玩家或房间创建者发出的、成功解析并接受的谁是卧底命令。旁观者、普通聊天、格式错误、重复描述和重复投票都不会延长房间或牌局的空闲期限。
_避免使用_：任意大厅消息、无效命令续命

**卧底监控快照**：
Web 可读取的脱敏谁是卧底状态，只包含模式、阶段、轮次、位置名单、存活状态、进度和剩余时间。它不包含词语、身份、私聊描述正文或具体投票人。
_避免使用_：秘密状态、完整牌局导出

**投票缺席提醒**：
秘密投票阶段每二十秒在大厅公布一次尚未投票的 A-K 位置，不公布已投位置、投票目标或实时票数。有人投票不会提前触发或重置提醒周期。
_避免使用_：完成进度批次、实时票数

**点歌帮助**：
由 `@帮助` 返回的点歌命令说明，只覆盖普通点歌、来源选择和 AI 点歌，不混入娱乐玩法说明。
_避免使用_：总帮助、全功能帮助

**娱乐帮助**：
由 `#娱乐`、`#帮助` 或保留入口 `@娱乐帮助` 返回的成语接龙、海龟汤、斗地主、跑得快和谁是卧底命令说明。
_避免使用_：点歌帮助、管理帮助

**同音接龙**：
由 `#同音接龙 成语` 开启的成语接龙模式。它比较首尾字的无声调拼音，不要求汉字或声调相同；开局后的普通提交、提示、解释、状态和结束仍属于同一局会话。本局使用过的成语永久禁止重复，提示只选择提交后仍有未使用后继的非死路成语。
_避免使用_：同字接龙、带声调接龙、谐音模糊匹配

**成语解释**：
从项目词库读取指定成语或当前接龙成语的来源与解释。词库只用前两个半角冒号分隔字段，缺失字段显示“未注明”；来源和解释分别按游戏聊天长度编号分段，并作为正式任务复用帮助消息的独占批量发送路径。
_避免使用_：在线词典、AI解释、接龙提示

**海龟汤会话**：
从题目被标记为已使用开始，到汤面公布、提问裁决、汤底结算全部结束为止的单局内存状态。程序重启不续局，进行中的汤底不会出现在 Web 快照中。
_避免使用_：AI 对话、题库记录、娱乐任务

**海龟汤使用记录**：
永久记录已经被选中过的题目 ID 的独立状态文件。题目会在汤面发送前写入，程序不提供重置接口；它不保存当前会话，也不表示题目已经成功结算。
_避免使用_：海龟汤会话、题库、运行状态文件

**海龟汤裁决**：
独立 AI Provider 对一条 `#` 提问给出的固定类别结果。完全正确需要第二次独立复核；游戏内只发送固定短句，不发送模型自由解释。
_避免使用_：候选歌曲审核、点歌 AI、自由对话

**待执行任务队列**：
主业务线程尚未执行的高层任务列表，和音乐播放队列不是同一个概念。普通聊天、稳定海龟汤提问、监听切换、启动任务和远程高层意图都通过类型化 `FormalTaskClient`/`FormalTaskSubmission` 进入正式 FIFO；队列只管理任务等待顺序、正式任务去重、执行线程唤醒和可观察标签，不承担任务执行语义，也不跨程序重启持久化。远程指挥台可以查看任务生命周期，并取消尚未开始的任务。
_避免使用_：点歌队列、HTTP请求队列

**模块业务意图**：
命令路由器、HTTP 适配器或运行时事件提交给一个纵向业务模块的类型化输入。中央路由只区分目标模块，具体意图变体由模块自己拥有。
_避免使用_：统一 UserCommand、聊天文本命令、UI 请求

**调度通道**：
业务调度器按现有优先级管理的正式任务、延迟发送和诊断三类工作来源。通道只影响何时开始；工作开始后仍遵循自身 UI 占用策略。
_避免使用_：线程、模块优先级、UI 中断

**待执行范围去重**：
待执行任务队列内部对已经排队但尚未执行的同语义任务做重复拒绝。它不判断命令是否来自旧 OCR、业务请求是否成立、当前歌曲是否应被保护，或管理投票是否已经在进行。
_避免使用_：命令屏幕锁、点歌互斥、业务有效性校验

**任务顺位恢复**：
执行线程已经取出任务后，因为暂停、界面未就绪或目标窗口暂不可用而把同一任务放回队首，表示恢复它原本的执行顺位。它不是普通入口提交新任务时使用的插队能力。
_避免使用_：远程插队、优先任务、强制抢占

**任务来源标签**：
待执行任务携带的可观察来源说明，用于日志、TUI 和远程指挥台展示。队列可以展示来源标签，但不能按来源改变入队顺序、去重规则或执行权限。
_避免使用_：来源权限、队列优先级、控制台特权

**任务等待状态**：
待执行任务队列中是否还有任务正在等待执行。它不同于执行线程是否正在处理任务，不能用来单独判断系统是否空闲。
_避免使用_：命令执行状态、执行器忙碌状态

**聊天监控区**：
远程指挥台中展示最新 OCR 聊天并提交控制台发言的区域。它把观察聊天上下文和发送控制台发言放在同一个操作位置。
_避免使用_：纯 OCR 面板、独立模拟聊天框

**监控刷新**：
远程指挥台自动获取最新运行状态的节奏，默认接近实时但必须允许暂停，避免远程查看日志或复制内容时界面持续跳动。
_避免使用_：手动刷新监控、不可暂停轮询

**监控快照**：
远程指挥台和 TUI 读取的内存态运行摘要，包含最近日志、最新 OCR、音乐播放队列、待执行任务标签和服务状态。它不是持久化状态，也不是游戏截图流。
_避免使用_：运行状态文件、实时截图、日志文件

**监控投影**：
根据各运行时和业务模块发布的类型化状态事件维护监控快照的单一所有者。生产模块不直接修改监控对象，Web 与 TUI 只读取不可变快照。
_避免使用_：共享 Monitor 锁、业务状态仓库、事件日志

**当前配置模板**：
项目唯一支持的完整配置 schema。所有当前字段必须显式存在并在任何运行时启动前完成模块校验；旧配置需要由使用者按模板重新填写，不在程序内迁移或补默认字段。
_避免使用_：配置迁移、兼容默认、隐式升级

**性能日志**：
单独记录阶段耗时的日志流，用来分析 OCR、UI 检测、命令执行等耗时。它不属于 TUI/Web 的事件日志。
_避免使用_：普通日志、事件日志

**执行命令日志**：
记录用户命令最终被处理成什么动作的持久日志。它用于追踪业务结果，不等同于实时事件日志。
_避免使用_：事件日志、性能日志

**按需游戏截图**：
远程指挥台手动获取的一次性游戏画面，用于检查当前游戏界面细节。截图保持原始尺寸，只做轻度 JPEG 画质压缩，不进入常驻监控刷新。
_避免使用_：实时截图流、缩放预览截图

**管理员启动前置条件**：
程序必须由管理员权限启动。远程指挥台不负责检测、展示或修复管理员状态，也不提供窗口状态诊断入口。
_避免使用_：Web提权、远程管理员状态检查、远程窗口诊断

**开门**：
从游戏启动页进入实际游戏世界的流程，主要通过 OCR 点击“点击进入”完成。
_避免使用_：登录、启动游戏

**启动游戏任务**：
主业务队列中的高层任务，负责查找或启动官服/国际服游戏，并完成开门直到游戏主界面可用。
_避免使用_：启动千星流程、登录任务

**进入千星任务**：
主业务队列中的高层任务，负责从游戏主界面进入千星奇域大厅，完成后回到可继续执行业务命令的一级界面。
_避免使用_：启动游戏任务、进出千星循环、千星刷新

**启动并进入千星**：
远程指挥台提交的组合意图，语义上只是把启动游戏任务和进入千星任务按顺序加入待执行任务队列。
_避免使用_：单体启动千星流程、HTTP直接进入千星

**官服/国际服通用启动**：
窗口和启动路径同时考虑 `YuanShen.exe` 与 `GenshinImpact.exe`，但不处理 B 服 bilibili 登录窗口。
_避免使用_：全渠道启动、B服登录

**自定义工作流解释器**：
把用户聊天命令匹配到配置流程，并为流程提供确认窗口、变量上下文和步骤执行边界的业务层。
_避免使用_：原子动作层、脚本引擎

**工作流上下文**：
自定义工作流执行时用于渲染消息、目标和参数的当前请求信息，包含触发用户、来源、命令名和参数。
_避免使用_：全局配置、运行状态

**好友发言**：
向指定好友私聊发送的一条业务反馈，通常用于邀请结果或自定义流程结果。它不是大厅回复，也不是玩家命令。二级私聊来源不保存可长期重开的会话句柄；若自动回到该私聊或发送反馈失败，改为在当前大厅发送明确的异常信息。
_避免使用_：大厅回复、模拟玩家命令

**邀请确认**：
非公共大厅接受好友邀请前，由大厅成员在等待窗口内给出的同意或拒绝决策。超时默认同意。
_避免使用_：好友确认、普通命令确认

**管理投票**：
拉黑或屏蔽聊天操作执行前，由好友私聊投票形成的临时决策。它需要同时接收多个好友的私聊表决，不能停留在单个二级好友会话中。
_避免使用_：普通确认、命令屏幕锁

**管理执行**：
管理投票通过后，在主业务队列中实际操作游戏好友界面的拉黑或屏蔽聊天动作。
_避免使用_：后台投票、即时管理命令

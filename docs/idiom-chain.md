# 成语接龙

成语接龙是一个独立的聊天娱乐模块。它不读取截图、不操作游戏窗口，也不调用在线词典或 AI；主程序只负责把 `@接龙` 命令放进现有串行任务队列，并发送模块返回的短回复。

## 命令

大厅蓝字消息使用以下格式：

```text
@接龙 开始 画蛇添足
@接龙 足智多谋
@接龙 状态
@接龙 结束
```

`@成语接龙` 是 `@接龙` 的同义入口。默认只有开局玩家可以结束对局，且同一玩家不能连续接词。

## 合法成语

合法性以内置词库为准，而不是“恰好四个汉字”的猜测：

1. 输入会去掉空白和常见标点，剩余内容必须全部是汉字。
2. 规范化后的文本必须与词库中的某一条完全一致。
3. 下一个成语首字必须等于当前成语尾字。
4. 最近历史中的成语不能重复使用。

简体和繁体按词库精确匹配，不自动互转。若词库中没有某个异体或繁体写法，它不会被接受。

模块会预先索引每个首字可接的成语；提交一个没有任何未使用后继的成语时，机器人宣布“封龙”并结束本局。

## 配置

项目词库默认启用，`assets/idioms.txt` 包含 30,345 条成语，并会随发布包的 `assets/` 目录一起携带：

```yaml
idiom_chain:
  enabled: true
  history_limit: 200
  idle_timeout_seconds: 300
  allow_consecutive_player: false
  allow_anyone_stop: false
```

词表来自固定版本的 `pwxcoo/chinese-xinhua`，只提取其成语 `word` 字段。程序启动时读取并索引该文本文件；完整来源、筛选方式、哈希和 MIT 许可证见 [词库来源核验](research/idiom-lexicon-source.md) 与 [第三方声明](../THIRD_PARTY_NOTICES.md)。更新词表不需要重新编译 EXE，但发布时必须保留 `assets/idioms.txt`。

`idle_timeout_seconds` 为 `0` 时禁用超时；否则下一条接龙命令到来时会检查空闲时间，超时对局会被清除。对局状态是进程内临时状态，程序重启后不会恢复。

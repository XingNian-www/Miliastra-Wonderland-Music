# 成语接龙

成语接龙是一个独立的聊天娱乐模块。它不读取截图，也不调用在线词典或 AI；聊天扫描线程直接更新接龙状态，普通短回复进入低优先级延迟聊天队列。`@解释` 是稳定发送例外：它进入正式 `PendingTask`，由独占命令执行线程按 `@帮助` 的批量发送方式连续输出。

成语接龙与海龟汤共用全局娱乐互斥，同一时间只能运行一局娱乐玩法。封龙、主动结束、空闲超时、游戏窗口关闭或进入新大厅都会释放占用。

## 命令

大厅蓝字消息使用以下格式：

```text
@接龙 开始 画蛇添足
@接龙 足智多谋
@同音接龙 开始 画蛇添足
@提示
@解释 画蛇添足
@解释
@接龙 状态
@接龙 结束
```

`@成语接龙` 是 `@接龙` 的同义入口。`@同音接龙 开始 成语` 会开启忽略声调的同音模式；开局后可以继续用 `@接龙 成语` 或 `@同音接龙 成语` 提交。`@提示` 只推荐符合当前模式、本局未使用且提交后仍有后继的成语，不会提示立即封龙的死路，也不会自动提交；如果所有候选都是死路，会明确回复没有安全提示。默认只有开局玩家可以结束对局，且同一玩家不能连续接词。

`@解释 成语` 查询指定成语；不带参数的 `@解释` 查询当前接龙最后一个成语。程序先单独发送成语，再分别发送来源和解释。来源或解释超过游戏单条聊天长度时，会以 `来源1/N`、`解释1/N` 编号分段，不会直接截断。整组消息复用 `@帮助` 的 `reply_batch` 路径，在一次屏幕独占任务中按 `timing.command.help_batch_ms` 间隔连续发送，期间不会被其他延迟回复穿插。

## 合法成语

合法性以内置词库为准，而不是“恰好四个汉字”的猜测：

1. 输入会去掉空白和常见标点，剩余内容必须全部是汉字。
2. 规范化后的文本必须与词库中的某一条完全一致。
3. 普通模式要求下一个成语首字等于当前成语尾字。
4. 同音模式比较无声调拼音，例如不同声调的 `足（zu）` 与 `组（zu）` 可以相接。
5. 同一局内出现过的成语不能再次使用，不受最近历史保存数量影响。

简体和繁体按词库精确匹配，不自动互转。若词库中没有某个异体或繁体写法，它不会被接受。

模块会预先索引每个首字以及无声调首字拼音可接的成语；提交一个没有任何未使用后继的成语时，机器人宣布“封龙”并结束本局。拼音来自离线 `pinyin` 字典，不调用网络；多音字采用字典默认读音。

词库每行格式为 `成语:来源:解释`，只使用前两个半角 ASCII `:` 分隔。全角 `：` 和解释字段中后续出现的半角 `:` 都属于正文。缺少或留空的来源、解释会显示为“未注明”，不会导致词库加载失败。

## 配置

项目词库默认启用，`assets/idioms.txt` 当前包含超过 30,000 条成语及其来源、解释，并会随发布包的 `assets/` 目录一起携带：

```yaml
idiom_chain:
  enabled: true
  history_limit: 200
  idle_timeout_seconds: 300
  allow_consecutive_player: false
  allow_anyone_stop: false
```

词表来自固定版本的 `pwxcoo/chinese-xinhua`，使用成语的 `word`、`derivation` 和 `explanation` 字段。程序启动时读取并索引该文本文件；完整来源、哈希和 MIT 许可证见 [词库来源核验](research/idiom-lexicon-source.md) 与 [第三方声明](../THIRD_PARTY_NOTICES.md)。更新词表不需要重新编译 EXE，但发布时必须保留 `assets/idioms.txt`。

`history_limit` 只限制最近历史的内存长度，不会放宽本局重复检查。`idle_timeout_seconds` 为 `0` 时禁用超时；否则主扫描循环会定期清理空闲超时的对局并释放全局娱乐互斥。对局状态是进程内临时状态，程序重启后不会恢复。

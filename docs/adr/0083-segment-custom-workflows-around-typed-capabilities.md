# 自定义工作流按机械段和类型化能力步骤执行

自定义工作流仍是唯一可以组合原子动作的入口，但不再把包含高层能力的整份定义嵌套执行为一个 UI 例程。解释器先校验完整定义，再把连续机械步骤编译成不可插入的自定义动作计划；遇到 `send_friend_message`、`invite_user` 等类型化能力步骤时结束当前机械段，通过拥有该能力的模块提交意图，保存工作流进度，并在 UI 运行时之外等待带操作标识的结果后继续下一段。内置业务不能改用这套工作流引擎。接受能力步骤之间可以运行其他已调度工作的成本，是为了保留用户自定义高层能力，同时避免嵌套 UI 例程、业务等待占用窗口和自定义流程绕过类型化契约。
